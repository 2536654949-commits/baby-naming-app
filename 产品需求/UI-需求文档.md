## 1. 产品概述

### 1.1 产品定位
一款基于AI的移动端宝宝起名工具，采用"一客一码"付费授权模式。用户输入授权码激活后，填写宝宝信息，AI智能推荐5个精选名字并提供详细文化解析。

### 1.2 目标用户
- **准妈妈小林**：孕晚期，希望名字有文化内涵，不懂五行八字，预算有限
- **新手爸爸小王**：工作忙，追求效率，相信AI技术能力

### 1.3 设计风格调性
```
关键词：温馨、简约、专业、可信

风格方向：
- 柔和的色调，避免冰冷感
- 圆润的造型，传递亲和力
- 清晰的层级，减少认知负担
- 精致的细节，建立品质感
```

## 2. 页面结构与需求

### 2.1 授权页（首页）

**页面功能**：用户输入授权码进行激活验证

**核心元素**：
- 品牌Logo/图标（使用婴儿emoji或设计图标）
- 产品名称："AI宝宝起名"
- 产品Slogan："智能起名 · 寓意深远"
- 授权码输入框（16位，格式：BABY-XXXX-XXXX-XXXX）
- 激活按钮

**交互状态**：
| 状态 | 展示内容 |
|------|----------|
| 默认 | 输入框为空，按钮禁用 |
| 输入中 | 实时格式化显示（自动转大写、加连字符） |
| 验证成功 | 绿色对勾 + "激活成功"提示 + 自动跳转 |
| 验证失败 | 红色错误提示 + 保留输入内容 |
| 提交中 | 按钮显示loading状态 |

**辅助信息**：
- 错误提示文案（授权码无效、已过期、已被使用等）
- 底部帮助链接："还没有授权码？如何获取"

**设计建议**：
- 背景可使用渐变色或品牌色，营造温暖氛围
- 卡片式布局，聚焦用户注意力
- 输入框需要大且清晰，方便输入

---

### 2.2 起名表单页

**页面功能**：收集宝宝基本信息

**页面布局**：
- 顶部导航：返回按钮 + 剩余次数显示（如：剩余次数: 7/10）
- 表单区域
- 底部固定：开始起名按钮

**表单字段**：

| 字段 | 类型 | 必填 | 交互方式 | 默认值 |
|------|------|------|----------|--------|
| 姓氏 | 单行输入 | 是 | 普通输入框 | 空 |
| 宝宝性别 | 单选 | 是 | 大按钮（男/女）互选 | 无 |
| 出生日期 | 日期选择 | 否 | 日期选择器 | 当天 |
| 出生时间 | 时间选择 | 否 | 时间选择器 | 当前时间 |
| 特殊要求 | 多行输入 | 否 | 文本域 + 字数统计（200字） | 空 |

**交互要求**：
- 表单验证：未填写的必填项标红并滚动到该位置
- 日期时间：底部弹出选择器
- 字数统计：实时显示（如：20/200字）

**设计建议**：
- 表单项间距充足，避免拥挤
- 性别选择使用大按钮，方便点击
- 选填项标识清晰（如标注"选填"）

---

### 2.3 加载过渡页

**页面功能**：AI生成过程中的等待页面

**展示内容**：
- 品牌图标/插图（可设计AI思考的插画）
- 主标题："AI正在为您精心起名..."
- 动画效果：跳动的三点或旋转图标
- 循环提示文案：
  - "正在分析姓氏特点..."
  - "正在查阅诗词典故..."
  - "正在筛选最佳组合..."
  - "即将完成..."

**设计建议**：
- 简洁温馨，缓解等待焦虑
- 动画流畅，传递"正在工作"的感觉
- 可使用渐变背景增加层次

---

### 2.4 结果列表页

**页面功能**：展示AI生成的5个名字

**页面布局**：
- 顶部：返回按钮 + 成功提示"🎉 已为您生成5个好名字"
- 列表区域：5个名字卡片，垂直排列
- 底部：再来一组按钮

**卡片内容**：
```
┌────────────────────────┐
│                        │
│      李泽言            │
│      Lǐ Zé Yán        │
│                        │
│   ⭐⭐⭐⭐⭐ 95分      │
│   恩泽之言，润物无声   │
│                        │
│      [查看详情 →]     │
│                        │
└────────────────────────┘
```

**卡片元素**：
- 名字：大字显示，突出
- 拼音：小字，灰色
- 评分：星级 + 分数（如：⭐⭐⭐⭐⭐ 95分）
- 寓意摘要：一句话概括（30字以内）
- 操作按钮："查看详情"

**交互要求**：
- 卡片按评分从高到低排序
- 点击卡片进入详情页
- 支持滚动浏览

**设计建议**：
- 卡片间距充足，方便区分
- 名字要醒目，是视觉焦点
- 评分用金色星星，增加品质感

---

### 2.5 名字详情页

**页面功能**：展示单个名字的完整解析

**页面布局**：
- 顶部导航：返回按钮 + 收藏按钮
- 名字展示区：大字显示名字 + 拼音
- 评分卡片：综合评分（星级 + 分数）
- 内容区块：寓意解析、典故出处、五行分析、亮点说明
- 底部：分享按钮（V2.0功能，可先预留）

**内容区块**：

**📖 寓意解析**
- 详细解释名字的寓意（100字左右）
- 说明对孩子成长的美好期望

**📚 典故出处**
- 引用诗词古文
- 标注来源（如：取自《诗经·大雅》）
- 如有古注，可引用（如：郑玄注：泽，润也）

**☯️ 五行分析**（如果用户提供了出生时间）
- 每个字的五行属性（如：李(木) 泽(水) 言(木)）
- 五行相生相克分析
- 寓意说明

**✨ 亮点说明**
- 音律和谐：平仄相间，朗朗上口
- 字形美观：结构匀称，书写流畅

**设计建议**：
- 每个区块用图标 + 标题区分
- 引用内容可用特殊样式（如：背景色、左边框）
- 内容排版要易于阅读
- 五行可用不同颜色的标签展示（木-绿色、水-蓝色等）

## 3. 关键交互流程

### 3.1 激活流程
```
打开应用
  → 输入授权码
  → 点击激活
  → 验证中（按钮loading）
  → 成功：提示 + 自动跳转表单页
  → 失败：显示错误提示，保留输入
```

### 3.2 起名流程
```
表单页
  → 填写信息
  → 点击开始起名
  → 表单验证
  → 验证通过 → 进入加载页
  → 验证失败 → 标红错误项
  → AI生成中（3-10秒）
  → 自动跳转结果页
```

### 3.3 查看详情流程
```
结果列表
  → 点击卡片
  → 进入详情页
  → 查看完整解析
  → 返回列表 / 收藏 / 分享
```

## 4. 统一设计要求

### 4.1 导航模式
- 页面跳转使用左滑/右滑过渡动画
- 顶部统一使用返回按钮（←）
- 固定顶部栏显示关键信息（如：剩余次数）

### 4.2 操作反馈
- **点击反馈**：按钮按下有缩放或阴影变化
- **加载状态**：显示loading图标 + 提示文字
- **成功提示**：绿色图标 + 简短文案
- **错误提示**：红色图标 + 具体错误原因

### 4.3 表单规范
- **必填项**：红色星号(*)标注
- **选填项**：灰色文字标注"选填"
- **错误状态**：红色边框 + 错误提示
- **成功状态**：绿色边框 + 对勾图标

### 4.4 空状态
- 无收藏时：显示空状态插图 + 提示文字
- 无网络时：显示断网插图 + 重试按钮

---

## 5. 品牌元素

### 5.1 品牌色彩（建议方向）
- **主色**：温暖色调（珊瑚粉、桃粉等）- 传递温馨
- **辅助色**：清新色调（薄荷绿、天蓝等）- 传递专业
- **背景**：白色/浅灰 - 保持简洁

### 5.2 图形元素
- **Logo**：婴儿形象 + AI元素结合
- **插图**：可采用扁平化风格，柔和的圆角
- **图标**：圆润风格，统一视觉语言

### 5.3 文案风格
- 友好亲切，避免生硬
- 清晰简洁，易于理解
- 正面积极，传递希望

---

## 6. 特殊场景

### 6.1 授权码快过期
- 剩余次数不足3次时，顶部显示提醒
- "您的使用次数即将用完，请及时购买新的授权码"

### 6.2 网络异常
- 无法连接服务器时，显示友好提示
- "网络连接失败，请检查网络后重试"
- 提供"重新加载"按钮

### 6.3 收藏功能
- 点击收藏按钮，切换收藏状态
- 已收藏的名字用实心星星标识
- 收藏数据保存在本地

## 7. 响应式要求

### 7.1 设备适配
- 主要针对移动端（iOS & Android）
- 支持屏幕宽度：375px - 428px
- 考虑刘海屏安全区域

### 7.2 横屏处理
- 横屏时适当缩小字号和间距
- 限制表单高度，支持滚动
